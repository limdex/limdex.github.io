# Design: Replace SolidJS Codebase with Svelte

**Date:** 2026-05-07  
**Status:** Approved

## Summary

Replace the contents of `my_porto` (SolidJS + Vite + SCSS) with the contents of `porto` (SvelteKit + Tailwind CSS), preserving the git history of `my_porto` by adding a new migration commit on top.

## Source and Target

| | Source (to copy from) | Target (to replace) |
|---|---|---|
| **Path** | `C:\Users\LimX\Work\self\porto` | `C:\Users\LimX\Work\self\my_porto` |
| **Framework** | SvelteKit + Svelte 5 | SolidJS 1.9 |
| **Styling** | Tailwind CSS 4 | SCSS |
| **Build** | `@sveltejs/adapter-static` (SSG) | Vite SPA |
| **Routing** | SvelteKit file-based routing | Client-side only |

## Migration Steps

### 1. Stage deletion of all current files
Run `git rm -rf .` inside `my_porto` to stage removal of every tracked SolidJS file.

### 2. Copy Svelte files into my_porto
Use Robocopy to copy all contents of `porto` into `my_porto`, excluding:
- `.git` — preserve `my_porto`'s git history
- `node_modules` — will be reinstalled
- `.svelte-kit` — generated by SvelteKit at dev/build time
- `build` — generated output
- `.legacy-static` — archived files not relevant to the new repo

### 3. Stage and commit
Run `git add .` to stage all incoming Svelte files, then commit with a clear migration message.

### 4. Install dependencies
Run `npm install` to install the SvelteKit/Tailwind dependency tree.

## What Is Preserved

- `.git` directory and full commit history of `my_porto`
- The working directory path (`my_porto`)

## What Is Replaced

Everything else: all source files, config files, `package.json`, `tsconfig.json`, `vite.config.ts`, `index.html`, `public/`, `src/`.

## Out of Scope

- No content merging between the two codebases
- No renaming or restructuring of the Svelte source
- No changes to the `porto` source directory
